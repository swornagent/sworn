### `cmd/sworn` failure triage
- `TestDoctor*` — doctor reports `[ERROR] baton/rules/ README.md missing
  rules-index heading` and `[ERROR] status timestamps: 87 violations` (future-dated
  `last_updated_at` / `verifier_verdict_at` across release boards). Cause is the
  Baton v0.5.0 re-vendor + dogfooding board data drifting from what the doctor
  checks assert. Triage as **data/vendored-content drift vs doctor strictness** —
  decide per check whether doctor or the data is wrong.
- `TestRouteIntegration` — **passed in isolation**, failed in the full run. Test
  ordering / shared-process-global-state smell (Baton Rule 11). Investigate before
  dismissing as flake.
- `TestCmdRun_Parallel` — failed in full run; runs to completion in isolation.
  Same suspicion.

### Public-cut blockers (not test failures)
- README.md and AGENTS.md both claim "zero runtime dependencies." False now —
  `go.mod` requires anthropic-sdk-go, aws-sdk-go-v2, oci-go-sdk, genai, bubbletea,
  modernc.org/sqlite. Contradicts the dep-policy revision (minimal justified deps +
  ADR). Fix copy or land the justifying ADRs before public.
- `doctor` exits 1 on this repo (87 timestamp violations + legacy-Baton-splice
  warnings). Dogfooding-data drift, not a product bug, but it is a bad first-run
  signal if a user runs `sworn doctor` against our own repo.

### Release-safety gap
`release.yml` triggers on `v*` tag push with **no dependency on `ci.yml`**. A tag
on a red tree runs GoReleaser anyway. Recommend gating the release job on CI (or a
branch/tag protection rule) so a red tree cannot be released.

## Test plan — phases

### Phase 0 — Make the CI gate green (blocking, do first)
1. Fix `implement_test.go:284` (`_, err := Run(...)`) → `go vet ./...` exits 0.
2. Triage + fix the three failing test packages; separate product bugs from stale
   board data and vendored-content drift.
3. `gofmt -w .`; add `gofmt -l` (fail on output) and `go build ./...` to `ci.yml`.
4. Gate `release.yml` on CI green.

### Phase 1 — Build & artifact integrity
- `make build` clean.
- Cross-compile the GoReleaser matrix: `CGO_ENABLED=0 GOOS={linux,darwin}
  GOARCH={amd64,arm64} go build ./cmd/sworn` — all four (confirm modernc sqlite
  stays CGO-free).
- `goreleaser release --snapshot --clean` (install goreleaser first; absent
  locally). Validates archives, brew formula, docker manifests, changelog.
- Docker scratch image: `docker build` + `docker run ... version`.
- Static check: `file`/`ldd` → not a dynamic executable.

### Phase 2 — CLI contract smoke (fail-closed invariant)
Table-driven e2e test asserting **verdict AND exit code together** for all 24
verbs. version=0, help=0, unknown=64, verify unconfigured=2, missing artefact=2,
lint bad-args=64, lint trace malformed=1. Commit it; do not leave manual.

### Phase 3 — Provider integration (BYO-key, spends tokens)
Per driver (anthropic, bedrock, oci, gemini/genai, openai-compat): `verify` a
trivial spec+diff; assert PASS=0 / FAIL=1 / BLOCKED=2. Negatives: bad key → fail
closed non-zero, no panic; no key → 2. Security: confirm no API key / request body
/ model payload is ever logged.

### Phase 4 — End-to-end loop (`sworn run`, the MVP)
Throwaway git repo + one trivial slice → `sworn run` → assert
plan→implement→fresh-context verify→retry/escalate→gated merge refuses on non-PASS.
One green-path and one fail-closed-path walkthrough minimum.

### Phase 5 — Public-safety audit
- Fix "zero runtime deps" claim (or land ADRs).
- grep committed files for business/pricing/strategy/private-repo refs.
- Confirm the private-docs symlink is NOT packaged by
  GoReleaser or Docker.
- Secret scan; `.gitignore` covers config/keys; LICENSE present.

### Phase 6 — Release rehearsal
Mirror `release.yml` smoke against the snapshot tarball: untar → `sworn version` →
`sworn verify` stub. Pull GHCR image on a clean container, run it. Dry-run the brew
`test do` stanza.
