## Origin

v0.12.0 specified `{{project_context}}` as *"supplied by the engine from the repo's configuration."* Sworn shipped **detection** instead ŌĆö it sniffs marker files and infers the stack. Two problems that hid behind treating detection as equivalent to configuration:

1. **Detection can't see stakes.** It reads languages off the filesystem; it cannot know whether the system is live, whether real customers depend on it, or whether it holds money ŌĆö which is exactly what should move a security finding from advisory to blocking.
2. **A guess handed to the model as a fact is graded against as a fact.** The dogfooding project was a Next.js/TS frontend with a **Go backend on Postgres, real paying customers, financial data**; detection reported *"a Next.js and TypeScript monorepo"* ŌĆö it missed the backend (top-level `go/` module outside a recognised workspace dir) and could not express stakes at all. This is how `"the SwornAgent project (a Go CLI)"` once survived inside a TypeScript repo.

Baton v0.13.0 makes the project context a **declared, human-ratified, version-controlled record** and keys the security-review blocking threshold to its stakes. This is the sworn work to consume it.

## What Baton ratified (the contract you implement against)

- **`project-context-v1`** ŌĆö `context` (one-line stack) + `stakes` (`production`, `real_users`, `sensitive_data` [closed enum], `regulated`, `notes`) + `ratification` (elicited ŌåÆ ratified ŌåÆ durable, same as journeys).
- **Fail closed:** stakes that are **absent, or present on an unratified record, are treated as HIGH.** An unconfirmed model proposal must never be able to *lower* the bar.
- **`security-review` is stakes-keyed** and now separates `severity` (impact) from `blocking` (disposition); the verdict is derived (FAIL iff a finding is `blocking:true`). At high stakes a `medium` finding blocks; at low stakes it advises. The six prompts and `llm-check-report-v1` now share one severity vocab (`critical|high|medium|low|info`).
- **`{{project_stakes}}`** joins the shared user payload; an engine that infers rather than reads the record **must label it inferred**, and must treat unstated stakes as high.

## Sworn work

Re-vendor to Baton **v0.13.0** first (pin the tag+SHA, both embed roots), then:

1. **`.sworn/project.json`** ŌĆö the on-disk record, validated against `project-context-v1`.
   - **`sworn init`** elicits it: the model *drafts* `context` + `stakes` by reading the repo (it can see the stack, the auth code, the payment integrations, the PII-holding schemas) and writes `ratification.ratified: false` + `drafted_by: <model>`. It **cannot** self-ratify ŌĆö only a human sets `ratified: true`.
   - Validate on write; a malformed record is an init defect surfaced now.
2. **`llm-check` consumes it** ŌĆö fill `{{project_context}}` from `context` and `{{project_stakes}}` from `stakes`. This replaces the v0.12.0 follow-up (de-hardcode the one-line context string): sworn now reads a full record, not a config string, and passes stakes into the security-review payload so the prompt's stakes-keyed grading fires.
3. **Fail-closed stakes resolution** ŌĆö when `.sworn/project.json` is absent, unratified, or its `stakes` are missing, pass **HIGH** stakes to the payload (production+real_users+sensitive_data effectively true) and label the context **inferred**. Never let an unratified/absent record grade at low stakes.
4. **`sworn doctor`** ŌĆö flag an **inferred** (detection-derived) or **unratified** context as undeclared, with the remedy: run `sworn init` and have a human ratify `.sworn/project.json`. This is the visibility mechanism so a silent detection-guess can't masquerade as a declaration.
5. **Detection stays, as a *labelled* fallback ŌĆö and gets fixed.** Keep detection for the no-record case, but (a) mark its output inferred everywhere it surfaces, and (b) fix the miss that started this: a **top-level `go/` module** outside a recognised workspace dir is currently invisible; detection must see it.

## Acceptance

- `sworn init` on a fresh repo writes a schema-valid `.sworn/project.json` with `ratified:false`; a human editing it to `ratified:true` is honoured.
- `sworn llm-check --check security-review` on a **high-stakes** repo blocks a `medium` finding; on a **low-stakes** repo the same finding is advisory; with **no/unratified** record it blocks (HIGH default).
- `sworn doctor` reports an inferred/unratified context as undeclared.
- Detection sees a top-level `go/` module (the original miss).
